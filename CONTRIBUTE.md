# Contributing to WEBRTC-TIC_TAC_TOE

## Getting Started
- Fork the repository.
- Clone your fork locally.
- Create a new branch for your work.

## Coding Guidelines
- Follow the existing coding style.
- Add comments to explain your code.
- Make sure to test your changes before submitting a pull request.

## Submitting Changes
1. Commit your changes in your feature branch.
2. Push the branch to your fork on GitHub.
3. Create a pull request from your fork to the original repository.

## Reporting Issues
If you find any bugs or have suggestions, please create an issue in the GitHub repository.

## Code of Conduct
Be respectful and considerate of other contributors. For more details, see the [Code of Conduct](CODE_OF_CONDUCT.md).

By contributing to this project, you agree that your contributions will be licensed under the project's MIT license.
